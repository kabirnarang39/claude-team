# Maintainer FAQ

## What is the one-sentence pitch?

Anton is a local workflow dashboard that gives Claude Code a repeatable 12-agent engineering process.

## What should we not claim?

- Do not claim autonomous production deployment.
- Do not claim speed benchmarks without data.
- Do not claim integrations that are not in `mcp-registry.yaml`.
- Do not imply the demo output came from external users.

## What is the strongest differentiator?

Anton is Claude Code-native, local-first, inspectable, and editable. Workflows are YAML, roles are Markdown, state is SQLite, and the dashboard shows every agent result.

## What is the weakest current point?

Real-world proof. The repo needs external users, real demo videos, and bug reports before the claims should get bigger.

## What feedback should we ask for?

- Is the workflow model too rigid?
- Are role prompts easy to customize?
- Does the dashboard help trust agent output?
- Which MCP integrations are worth supporting first?

## What should be fixed before a large launch?

- Run through install from a clean macOS and Linux machine.
- Record a real workflow video, not only `anton --demo`.
- Add at least one real-world case study with permission.
