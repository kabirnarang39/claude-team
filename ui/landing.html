How it works

From task to team in one command.

Anton runs on your machine around your Claude Code session. The stack is plain Go, SQLite, YAML workflows, and Markdown role prompts.

Dispatch
Type one slash command in Claude Code. Describe the task in plain English; Anton routes it through the configured workflow.
/team-dispatch build Stripe billing
Execute
Anton dispatches 12 specialists across 5 phases. Planning runs first; engineering, QA, and security run in parallel where dependencies allow.
Inspect
A live DAG renders in your browser. Click any agent to see their output, token usage, and deliverables as they complete.

Capabilities

Built for real engineering work.

Live WebSocket dashboard
A real-time DAG visualization in your browser. Watch agents execute, inspect their output and token usage, and review deliverables as they're produced, without leaving your workflow.
Parallel execution
Phases gate on dependencies; agents within a phase run in parallel. Sequential and parallel modes configurable per workflow.
YAML workflows
Drop a .yaml into workflows/ to define custom agent pipelines for any use case.
Security built-in
The default workflow includes a dedicated security-review role. Findings are saved for review alongside QA and code-review artifacts.
Human review gates
Pause the pipeline at any checkpoint for human review before agents proceed to the next phase.
MCP native
Exposes a stdio MCP server. Agents coordinate via report, ask, and reply tools. No polling, no shared memory.
Free & open source

Make Claude Code workflows visible and repeatable.

Open source and local-first. Uses your Claude Code session; optional MCP tools use their own configured tokens.
